New 2026-05-13

One Konnect organization carries three control planes — one per environment. Three separate K8s scopes attach as data planes, each controlled through the IaC repo under PR-gated promotion. View 1 (below) shows the in-cluster shape that repeats inside each of these three K8s scopes.

Single Konnect org, three control planes. Modern Kong pattern replacing "Runtime Groups." Single billing, single audit trail, environments operationally isolated — a Dev config error cannot reach Prod data planes. Open — arch team Row 3
K8s scope per env is the platform team's choice. Two shapes: (a) separate namespaces on a shared cluster; (b) separate clusters per env. Option (b) is the stronger isolation story for Prod at Tier 2+. Open — arch team Row 1
Secrets are scoped per env. vault://dev/, vault://test/, vault://prod/ — no cross-env path collisions. Dev uses sandbox credentials; Prod uses production keys with strict rotation cadence.
Phase transitions are config-only, not infrastructure re-architecture. Phase 1 (observe) → Phase 2 (enforce) is a decK YAML delta promoted Dev → Test → Prod. No re-deploy. This is the property that makes the four-phase plan operationally cheap.
IaC repo is the only path to any env. No hand-edits in the Konnect UI in any environment. Prod is the hard rule; Dev / Test are operating discipline. Emergency hot-fixes get a 24h back-fill PR.
Network segmentation between envs is non-negotiable. Prod data planes never reach Dev / Test services or vice versa. Consumed from the DXC platform team's network-segmentation story — not invented by the gateway team.

Plugin Request Path — Recommended v1 Plugin Chain (Row 4)

Environment-agnostic: describes how a request traverses the plugin chain regardless of which env the route is in. Phase determines what plugins are on the chain (Phase 1 = File Log + OTel only; Phase 2 adds enforcement plugins); env determines which CP and which vault-scoped keys are in play. Order is load-bearing — policy violations do not consume token budgets; PII is redacted before leaving the cluster. Steps marked Row 6 conditional drop if compliance defers PII redaction past v1.
